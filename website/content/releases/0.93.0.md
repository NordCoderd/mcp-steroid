## Highlights

### Plugin Verifier Compliance

Six internal IntelliJ API usages replaced with public alternatives — down from 29 to just 4 unavoidable internal references. This improves forward compatibility with future IntelliJ versions.

### Rewritten MCP Server Instructions

Agents now see concise, exec_code-first instructions: MCP Steroid is a **stateful API** and **drop-in replacement for LSP**. Three-step getting started: `list_projects` → `fetch_resource` → `execute_code`.

### New Tool: `steroid_fetch_resource`

Load `mcp-steroid://` skill guides directly from the tool list. Task-aware description maps your intent to the right resource — running tests, debugging, or general IDE work.

### Maven Test Execution Fix

Fixed a critical bug where the ModalityStateMonitor killed Maven test execution prematurely. Maven + Gradle integration tests now validate end-to-end test execution in Docker.

### Improved Action Discovery

`steroid_action_discovery` now explains the discovery→exec_code workflow: find available quick-fixes and gutter actions first, then invoke them programmatically.

### CI and Infrastructure

- GitHub Actions Node24 compliance across all workflows
- TeamCity artifact post-processing improvements
- Docker shell quoting fixes for bind-mount scenarios
- Agent model updates: Codex gpt-5.4, Gemini gemini-3.1-pro-preview
