## 0.91.0

### Compilation Error Visibility Fix

**AI agents can now see full compilation error details.** Previously, when code failed to compile, agents like Claude Code only saw "Execution ID: eid_..." — the actual compiler error message (type mismatches, unresolved references, syntax errors) was invisible. This made it impossible for agents to diagnose and fix compilation failures.

**Root cause:** The MCP tool result contained multiple separate text content items, with the execution ID as the first entry. Some MCP clients only render the first text item when `isError=true`, hiding all subsequent content including the compiler output.

**Fix:** Tool results now always merge all text into a single content block, ensuring the complete output — compiler errors, warnings, hints — is visible to every MCP client. ([#14](https://github.com/jonnyzzz/mcp-steroid/issues/14))

### Progress Reporting Fix

MCP progress notifications were silently discarded because `ExecuteCodeToolHandler` was not passing the progress reporter through to the execution engine. Agents using `_meta.progressToken` now receive real-time progress updates during code compilation and execution.

The `mcpProgressReporter` parameter on `ExecutionManager.executeWithProgress()` is now mandatory (no default value), preventing this class of bug from recurring.

### Other Improvements

- **Cleaner tool output:** Removed duplicate "Execution ID:" line and "use it to report feedback" noise from tool responses.
- **EULA fix:** The plugin build and GitHub releases now correctly bundle the EULA instead of the Apache 2.0 license.
- **Test infrastructure:** Added `AiProcessResult` type with `rawStdout` for accessing unfiltered NDJSON output. Made `ProcessResult` extension functions generic to preserve concrete types through chaining.
- **Compilation error/warning tests** now run for all three agents (Claude, Codex, Gemini) via `CliIntegrationTestBase`.
- **Website restructure:** The `website/` folder is now a separate repo clone, gitignored from the main repository.
